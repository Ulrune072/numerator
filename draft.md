## Ядро (обязательно для запуска)

### Авторизация
- Страница входа и регистрации — первый экран для всех
- Форма: email + пароль, кнопка «войти» / «создать аккаунт»
- После регистрации автоматически создаётся запись в `profiles`
- Забытый пароль — через Supabase Auth (письмо на email)
- Нет авторизации → нет доступа ни к одной странице

### Лекции
- 5 тем на старте (примерный список ниже)
- Каждая лекция — отдельная страница с текстом, примерами, таблицами
- Внизу лекции кнопка «Перейти к заданиям» → `/tasks/[slug]`
- Лекции отображаются в виде карточек на `/lectures` — название, краткое описание, статус (пройдено / не пройдено)
- Порядок лекций фиксированный (по `order_index`)

**Примерные 5 тем:**
1. Количественные числительные (один, два, сто, тысяча...)
2. Порядковые числительные (первый, второй, сотый...)
3. Склонение числительных по падежам
4. Числительные в составе словосочетаний (согласование с существительными)
5. Дробные и собирательные числительные

### Задания
- После каждой лекции — тест из 5–10 вопросов
- Формат: один вопрос на экране, 4 варианта ответа
- После ответа — мгновенная обратная связь: правильно ✅ / неправильно ❌ с пояснением
- По завершении теста — экран результатов: сколько баллов получено, сколько правильных ответов
- Повторное прохождение разрешено, но баллы не начисляются повторно

### Личный кабинет (`/dashboard`)
- Имя пользователя (из профиля)
- Текущий статус/номинация (Новичок → Числитель)
- Прогресс-бар до следующего статуса
- Список тем: пройдено / не пройдено
- Общий счёт баллов

### Диагностический тест
- Отдельная страница `/diagnostic`
- Google Form встроена через iframe
- Текст-подводка: «Пройди тест, чтобы узнать свой уровень»
- Результат теста не влияет на баллы (только диагностика)

### Админ-панель (`/admin`)
- Вход только для пользователей с `role = 'admin'`
- Управление лекциями: создать / редактировать / удалить / опубликовать
- Управление заданиями: добавить вопрос к лекции, указать варианты, отметить правильный
- Список пользователей с их баллами и прогрессом (просмотр, без редактирования)

## Идеи для развития (после запуска)

### UX / геймификация
- Анимация при получении нового статуса (конфетти, звук)
- Стрик — количество дней подряд, когда пользователь заходил на сайт
- Таблица лидеров — топ-10 по баллам (опционально, может демотивировать)
- Достижения-бейджи помимо статусов (например: «Первая лекция», «Все темы пройдены»)

### Контент
- Поиск по лекциям
- Печать / экспорт лекции в PDF
- Закладки — пользователь может сохранить лекцию
- Связанные темы — ссылки между лекциями

### Задания (расширенные форматы)
- Задания с вводом текста (не только multiple choice)
- Упражнения на заполнение пропусков в предложении
- Аудио-задания (прослушать и выбрать правильную форму числительного)
- Таймер на прохождение теста

### Технические улучшения
- Тёмная тема
- Мобильная адаптация (responsive) — важно для школьников
- Offline-режим (PWA) — чтение лекций без интернета
- Email-уведомления о новых лекциях

### Аналитика для администратора
- Какие вопросы вызывают больше всего ошибок
- Среднее время прохождения теста
- Воронка: лекция просмотрена → тест начат → тест завершён

## Открытые вопросы

- Нужна ли возможность регистрации через Google (OAuth)? — пока нет, но Supabase поддерживает
- Как хранить контент лекций — Markdown в БД или HTML? — склоняюсь к Markdown + рендеринг через `react-markdown`
- Нужен ли reset прогресса для пользователя? — возможно, в настройках профиля
- Интернационализация (i18n) — интерфейс только на русском или ещё на английском? — пока только русский
- Нужна ли роль «учитель» (промежуточная между студентом и администратором)? — рассмотреть позже
